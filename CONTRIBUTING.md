# Guía de contribución

¡Gracias por tu interés en contribuir a este proyecto! 🙂

Este repositorio tiene un enfoque de aprendizaje, por lo que son bienvenidas las contribuciones orientadas a mejorar la claridad, la organización del código o a añadir nuevos ejemplos de práctica.

## Cómo contribuir

1. Haz un fork del repositorio

2. Crea una nueva rama a partir de `main`(desde terminal en Linux/macOS):
   ```bash
   git checkout -b feature/nombre-de-tu-feature

3. Realiza tus cambios con commits pequeños y descriptivos

4. Sube tu rama a tu fork

5. Abre una Pull Request explicando:
    - Qué cambios has realizado.
    - Por qué los has realizado.

## Ideas para contribuir

- Mejorar la documentación o la claridad del README

- Añadir nuevas consultas SQL o ejemplos de JOIN

- Incluir consultas con funciones de agregación (COUNT, SUM, etc.)

- Mejorar la validación de datos o el manejo de errores

- Añadir comentarios que ayuden a entender mejor el código

## Reglas básicas

1. Mantén el código simple y legible

2. Sigue la estructura del proyecto

3. Añade comentarios si el cambio lo requiere

4. No subas archivos generados (por ejemplo .db si no es necesario)

## ¿Dudas?

Si tienes cualquier pregunta, puedes abrir un issue o contactar conmigo directamente.

¡Gracias por contribuir! =D
